# User Stories & Acceptance Criteria
**Project:** MCE Platform
**Date:** 2025-12-28
**Source:** Derived from [Product Requirements Document](Product_Requirements_Document.md)

## 1. User Roles
*   **Sales Representative:** Primary user. Focuses on responding to leads and converting them.
*   **Admin/Manager:** Focuses on system configuration, campaign setup, and performance tracking.
*   **System (Bot):** Automated agent handling ingestion and initial trigger-based responses.

## 2. Lead Management

### Story 2.1: Centralized Lead View
**As a** Sales Representative,
**I want** to see all incoming leads from various sources (CSV, Sheets, Webhooks) in a single real-time list,
**So that** I don't have to switch between tabs/apps to check for new potential customers.

**Acceptance Criteria:**
- [ ] Dashboard displays a "Live Feed" list of leads.
- [ ] List includes Lead Name, Status, Score, and Timestamp.
- [ ] New leads appear instantly without page reload (auto-polling/socket).

### Story 2.2: Lead Detail Access
**As a** Sales Representative,
**I want** to click on a lead to see their full details and history,
**So that** I have context before reaching out.

**Acceptance Criteria:**
- [ ] Clicking a lead opens a detailed view/card.
- [ ] Displays source, phone number, and current status.
- [ ] Shows history of previous messages/interactions if available.

### Story 2.3: Bulk Import
**As an** Admin,
**I want** to upload a CSV file of offline leads,
**So that** they can be entered into the system for processing and engagement.

**Acceptance Criteria:**
- [ ] UI provides a "Upload CSV" button.
- [ ] System parses the file and creates Lead records.
- [ ] Duplicates (by phone number) are handled gracefully (skipped or merged).

## 3. Communication

### Story 3.1: WhatsApp Integration
**As a** User,
**I want** to link my WhatsApp account via QR code,
**So that** the system can send and receive messages using my business number.

**Acceptance Criteria:**
- [ ] System displays a QR code if not authenticated.
- [ ] Status updates to "Ready" or "Connected" once scanned.
- [ ] Session persists across server restarts.

### Story 3.2: Direct Messaging
**As a** Sales Representative,
**I want** to send a WhatsApp message to a lead directly from the dashboard,
**So that** I can initiate a conversation immediately.

**Acceptance Criteria:**
- [ ] Lead detail view has a "Chat" interface.
- [ ] Typed messages are sent via the connected WhatsApp session.
- [ ] Sent messages appear in the chat history with accurate status (sent/delivered).

## 4. Automation & Workflows

### Story 4.1: Auto-Reply to New Leads
**As a** Manager,
**I want** the system to automatically send a welcome message to any new lead,
**So that** engagement happens instantly (0-minute response time) even if I'm busy.

**Acceptance Criteria:**
- [ ] Workflow trigger "On Lead Created" detects new entries.
- [ ] System sends a pre-defined template message.
- [ ] Action is logged in the lead's history.

### Story 4.2: Google Sheet Sync
**As a** Marketer,
**I want** the system to automatically pull new rows from my Facebook Ad leads sheet,
**So that** my paid traffic is funneled into the sales dashboard without manual export/import.

**Acceptance Criteria:**
- [ ] System polls the configured Google Sheet URL at set intervals.
- [ ] New rows are converted to Lead records.
- [ ] System tracks which rows have already been processed to avoid duplication.

## 5. Analytics & Insights

### Story 5.1: Performance Overview
**As a** Manager,
**I want** to view high-level metrics like Total Leads and Conversion Rate,
**So that** I can assess the health of my sales pipeline.

**Acceptance Criteria:**
- [ ] Dashboard header displays key counters.
- [ ] "Performance Chart" visualizes lead volume trends over time.
